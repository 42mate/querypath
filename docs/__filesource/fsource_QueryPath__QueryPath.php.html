  1. <?php
  2. /**
  3.  * The Query Path package provides tools for manipulating a Document Object Model.
  4.  * The two major DOMs are the XML DOM and the HTML DOM. Using Query Path, you can
  5.  * build, parse, search, and modify DOM documents.
  6.  *
  7.  * To use Query Path, this is the only file you should need to import.
  8.  *
  9.  * Standard usage:
  10.  * <code>
  11.  * <?php
  12.  * $qp = qp('#myID', '<?xml version="1.0"?><test><foo id="myID"/></test>');
  13.  * $qp->append('<new><elements/></new>')->writeHTML();
  14.  * ?>
  15.  * </code>
  16.  *
  17.  * The above would print (formatted for readability):
  18.  * <code>
  19.  * <?xml version="1.0"?>
  20.  * <test>
  21.  *  <foo id="myID">
  22.  *    <new>
  23.  *      <element/>
  24.  *    </new>
  25.  *  </foo>
  26.  * </test>
  27.  * </code>
  28.  *
  29.  * To learn about the functions available to a Query Path object,
  30.  * see {@link QueryPath}. The {@link qp()} function is used to build
  31.  * new QueryPath objects. The documentation for that function explains the
  32.  * wealth of arguments that the function can take.
  33.  *
  34.  * Included with the source code for QueryPath is a complete set of unit tests
  35.  * as well as some example files. THose are good resources for learning about
  36.  * how to apply QueryPath's tools.
  37.  *
  38.  * If you are interested in building extensions for QueryParser, see the
  39.  * {@link QueryPathExtender} class. There, you will find information on adding
  40.  * your own tools to QueryPath.
  41.  *
  42.  * QueryPath also comes with a full CSS 3 selector parser implementation. If
  43.  * you are interested in reusing that in other code, you will want to start
  44.  * with {@link CssEventHandler.php}, which is the event interface for the parser.
  45.  *
  46.  * If you want to learn the nitty gritty details of QueryPath, you can take a
  47.  * look at the implementation in {@link QueryPathImpl.php}. There you will find
  48.  * the "real" code.

  80. /**
  81.  * Build a new Query Path.
  82.  * This builds a new Query Path object. The new object can be used for
  83.  * reading, search, and modifying a document.
  84.  *
  85.  * While it is permissible to directly create new instances of a QueryPath
  86.  * implementation, it is not advised. Instead, you should use this function
  87.  * as a factory.
  88.  *
  89.  * Example:
  90.  * <code>
  91.  * <?php
  92.  * qp(); // New empty QueryPath
  93.  * qp('path/to/file.xml'); // From a file
  94.  * qp('<html><head></head><body></body></html>'); // From HTML or XML
  95.  * qp(HTML_STUB); // From a basic HTML document.
  96.  * qp(HTML_STUB, 'title'); // Create one from a basic HTML doc and position it at the title element.
  97.  *
  98.  * // Most of the time, methods are chained directly off of this call.
  99.  * qp(HTML_STUB, 'body')->append('<h1>Title</h1>')->addClass('body-class');
  100.  * ?>
  101.  * </code>
  102.  *
  103.  * This function is used internally by QueryPath. Anything that modifies the
  104.  * behavior of this function may also modify the behavior of common QueryPath
  105.  * methods.
  106.  *
  107.  * @param mixed $document 
  108.  *   A document in one of the following forms:
  109.  *   - A string of XML or HTML (See {@link HTML_STUB})
  110.  *   - A path on the file system
  111.  *   - A {@link DOMDocument} object
  112.  *   - A {@link SimpleXMLElement} object.
  113.  *   - A {@link DOMNode} object.
  114.  *   - An array of {@link DOMNode} objects (generally {@link DOMElement} nodes).
  115.  *   - Another {@link QueryPath} object.
  116.  *
  117.  *  Keep in mind that most features of QueryPath operate on elements. Other
  118.  *  sorts of DOMNodes might not work with all features.
  119.  * @param string $string 
  120.  *   A CSS 3 selector.
  121.  * @param array $options 
  122.  *   An associative array of options. Currently supported options are:
  123.  *   - <none> (Coming soon: DOM parser settings.)
  124.  */

  429.    *   special treatment of $this, as there is in jQuery.)

  431.    *     object (DOMNodes are all objects).

  433.    *   works like break in a standard loop.

  435.    * - All other return values are ignored.
